# 业务操作不可用状态防错规则

> 关联业务：管理端对表决、审核、合同、验收等受角色、状态、资料和期限约束的操作，必须给经办人明确且可行动的办理提示。

## 1. 重复问题根因

仅把按钮设为不可点击，会把角色限制、资料缺失、状态不符和时间不符混成同一种视觉结果。经办人无法判断是自己没有权限、表单未填完整，还是业务规则不允许，容易把正常规则误认为系统故障。

时间输入还有一个常见边界：服务端时间包含秒和毫秒，`datetime-local` 通常只填写到分钟。直接截断会使页面显示的时间仍早于服务端允许时点，导致用户按提示填写后仍无法提交。

## 2. 实现规则

1. 不可点击按钮旁必须显示当前最先需要处理的一条业务原因，不能只依赖灰色、悬停提示或接口报错。
2. 身份不符时说明谁可以办理；资料缺失时指出缺少的材料；期限不符时同时显示规则允许的最早时间。
3. 页面不得通过放宽后端校验解决可用性问题。权限、状态和期限仍以服务端为准。
4. 服务端时间进入分钟精度输入框时必须向上取整，并设置输入框最小值，不能向下截断。
5. 每个受多重条件控制的关键按钮，应把不可办理原因抽成可测试函数，并至少覆盖有权且资料完整、时间过早、身份不符三种场景。

## 3. 发布检查

- 用有权身份验证资料完整且时间合规时按钮可点击；
- 用无权身份验证页面显示办理主体，不暴露英文权限错误；
- 用早于规则的时间验证页面给出准确日期，且不会发出请求；
- 用等于页面建议的分钟验证不会因服务端秒数精度再次失败。
